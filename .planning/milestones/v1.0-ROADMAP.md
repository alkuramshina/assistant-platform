## Milestone Summary

v1.0 shipped an SSH-deployed multi-bot console that can be installed on a customer-provided Linux VM, create isolated Telegram Nanobot instances, store secrets server-side, start/stop bots, and inspect activity/runtime logs.

Key decisions:

- `README.md` is the durable product/infrastructure source of truth.
- `.planning/` is working memory and phase history.
- Docker Compose is the per-bot runtime template.
- Bot containers do not mount `docker.sock`.
- Telegram allowlist is mandatory from first bot deployment.
- Current UI targets OpenRouter model presets; broader provider selection is future work.

Human verification:

- Phase 9 Telegram smoke and Activity logging passed on 2026-04-21.
- Phase 10 two-bot isolation UAT passed on 2026-04-21.

Deferred:

- Web console authentication/protection.
- Full provider selector beyond OpenRouter presets.
- More polished domain/TLS lifecycle guidance.
